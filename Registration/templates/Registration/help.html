Description on how this project is created

              Initially one folder is created in which we are expecting to store all files and folders of Django project. By using command-prompt this newly created folder is accessed. Initially by typing python at command-prompt python is checked to be present on selected drive. Another way is to type python --version so version of python is displayed on command-prompt. Afterwords we have to create virtual envioronment so that all installations resides in the virtual environment so created. The command-prompt to create virtual envioronment is virtualenv venv. The folder venv is created inside the folder at which command is running. To activate the virtual environment use venv\Scripts\activate. Now to check the django version present we have to execute django-admin --version

Django project creation

             Now we will use the command django-admin startproject Engg_Inst_Site to start new project. Generally one or more applications are created inside the project, for creating application we have to go inside the the folder Engg_Inst_Site by using cd Engg_Inst_Site. Now we may check for django installation by using python -m django --version. If django is not present then use pip install django to install django. The name of the project to created is decided as Registration, so we have to use python manage.py startapp Registration.Now to view the file structure created while creating django project and application we have to go one level up by using cd.. so we are at root folder that we have created at the beginning. Now if we use code . command then visual code window gets open and we see all folder and file structure so created.

             Then we may type in python manage.py runserver at terminal to start webserver. Once webserver is started then it is ready to listen the request at http://127.0.0.1:8000/ . So we write the http://127.0.0.1:8000/ in address bar of browser and djangoâ€™s default page is displayed.

For this change to implement we have to modify settings file, we will either replace existing database part of settings file or comment out the earlier part and add new part corresponding to database
